My Work

Z80 Processor Simulator

I developed a functional simulator of the Z80 microprocessor using Java, designed as an educational tool to help students understand low-level computer architecture and instruction execution. The project replicates key components of the processor, including the Arithmetic Logic Unit (ALU), control unit, registers, instruction set, and memory management. The simulator was implemented using object-oriented programming principles and data structures to efficiently represent memory, registers, and instruction handling. I designed the architecture to emulate the fetch–decode–execute cycle, allowing instructions to be interpreted and executed similarly to how the real Z80 processor operates. To enhance usability and debugging capabilities, I built a graphical interface using Java Swing that displays processor registers, memory states, and instruction flow in real time. The system also includes step-by-step execution and debugging features, enabling users to analyze how each instruction affects memory and register values. Concurrency was implemented through multithreading to manage execution flow and interface updates simultaneously, improving performance and responsiveness of the simulator. The simulator is currently used by students in the Computer Structure and Programming course as a learning tool to better understand processor behavior, assembly instructions, and low-level system operation

Java, Computer Architecture, CPU Simulation, Data Structures, OOP, Java Swing

Relational Database Design and Implementation

I designedI designed and implemented a relational database system using Microsoft SQL Server, covering the main stages of database development from conceptual design to implementation. The project started with the creation of an Entity–Relationship (ER) model to identify entities, attributes, and relationships within the system. After defining the conceptual model, I translated it into a logical relational schema, applying normalization up to Third Normal Form (3NF) to eliminate redundancy and maintain data integrity. I then implemented the database using Transact-SQL (T-SQL), creating tables, constraints, and relationships using primary and foreign keys. To support data operations and improve maintainability, I developed stored procedures, triggers, views, and functions that automate processes and enforce business rules directly at the database level. I also optimized query performance by implementing indexes and efficient SQL queries, including joins, subqueries, and aggregation functions.

SQL Server, T-SQL, Database Security, Data Integrity, Stored Procedures, Indexing
